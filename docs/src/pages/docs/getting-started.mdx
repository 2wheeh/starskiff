# Getting Started [Install starskiff and boot your first Cosmos SDK node]

## Overview

starskiff spawns **real Cosmos SDK nodes** as child processes for integration tests — no Kubernetes, just the chain's node from a binary or a container image. An instance boots in seconds: `init` → genesis patching → account funding → `gentx` → `start` → health-polled until the first block.

:::warning
starskiff is under active development. APIs may change without notice until v1.0.
:::

## Installation

:::code-group

```bash [pnpm]
pnpm add -D starskiff
```

```bash [npm]
npm install -D starskiff
```

```bash [yarn]
yarn add -D starskiff
```

```bash [bun]
bun add -D starskiff
```

:::

## Prerequisites

Most instances run from an image, so a running **Docker** is all you need — `simd`, `wasmd`, and `xplad` pull their image on first use (`evmd` too, once its [image is published](/docs/guides/docker#images-we-publish); until then run it from a binary).

A few chains are binary-backed (`gaiad`, `hermes`, and the private `marood`) — put their executable on `PATH`, e.g. from an official release:

```bash
gh release download v27.5.0 --repo cosmos/gaia --pattern "gaiad-v27.5.0-linux-amd64" --dir /tmp
install -m755 /tmp/gaiad-v27.5.0-linux-amd64 ~/go/bin/gaiad
```

Any instance also accepts a `binary` or `image` override — the [escape hatch](/docs/guides/docker#escape-hatches) for local development. See the [instance pages](/docs/instances) for the full list, and the [CI guide](/docs/guides/ci) for provisioning binaries on CI runners.

## Quick Start

```ts
import { Instance, testAccounts } from 'starskiff';

const [alice] = testAccounts;

const instance = Instance.simd({
  chainId: 'test-1',
  accounts: [{ mnemonic: alice.mnemonic, coins: '1000000000stake', name: alice.name }],
});

await instance.start();

// Ready-to-use endpoint URLs:
instance.rpcUrl; //  http://localhost:26657 — CometBFT RPC
instance.grpcUrl; // http://localhost:9090  — gRPC
instance.apiUrl; //  http://localhost:1317  — REST (Cosmos SDK API)

await instance.stop();
```

`start()` resolves once the node reports its first block (and, for EVM chains, once the JSON-RPC server responds). `stop()` kills the process and removes the temporary home directory — every run starts from a fresh genesis.

## Using with cosmjs

```ts
import { SigningStargateClient, GasPrice } from '@cosmjs/stargate';
import { DirectSecp256k1HdWallet } from '@cosmjs/proto-signing';

const wallet = await DirectSecp256k1HdWallet.fromMnemonic(alice.mnemonic, { prefix: 'cosmos' });

const client = await SigningStargateClient.connectWithSigner(instance.rpcUrl, wallet, {
  gasPrice: GasPrice.fromString('0stake'),
});
```

For wiring instances into your test runner, see the [vitest guide](/docs/guides/vitest); for EVM JSON-RPC testing, see the [evmd instance](/docs/instances/evmd).
